How the valuation layer works

No universal semiconductor multiple: each company is assigned a framework based on its economics. EDA/IP is evaluated with recurring-growth metrics; memory and storage with forward/normalized cycle metrics; foundries with capital intensity and cash returns; distressed names with enterprise value and liquidity.

Expectations burden: a 1–10 model-derived indicator. Higher means the live valuation appears to require more growth, better margins or a more favorable cycle outcome. It uses the relevant forward multiple, growth, historical valuation position and estimate revisions when available. It is not a price target.

Revision momentum: the updater stores one daily consensus snapshot. The 30- and 90-day revision fields remain blank until enough history has accumulated. This is intentional; the site never backfills invented revisions.

Live-data architecture: GitHub Pages stays static and public. A scheduled GitHub Action reads a private API key from GitHub Secrets, updates JSON files, validates them, and commits only the generated data. No API key is shipped to the browser.

Fallback behavior: if the market-data provider fails or is not configured, all 57 issuer-sourced research profiles remain available and the site clearly labels the live layer as unavailable or stale.

Enable automatic updates

  1. Create an account with the supported market-data provider and obtain an API key with access to the endpoints you need.
  2. In GitHub: Settings → Secrets and variables → Actions → New repository secret.
  3. Name the secret FMP_API_KEY and paste the key. Never place the key in app.js or any public file.
  4. Open Actions → Update semiconductor market data → Run workflow and select a deep refresh for the first run.
  5. The workflow then runs hourly for quotes and refreshes the heavier fundamentals/consensus layer approximately daily.
